# YouTube Download — Telegram-бот

Бот для скачивания видео и аудио с YouTube по ссылке: выбор «видео / только аудио», качество, отправка файла в чат. Служебные сообщения удаляются, остаются только результат и ссылка на источник.

## Требования

- **Python 3.10+**
- **FFmpeg** — обязателен (объединение потоков и извлечение аудио)
- Токен бота от [@BotFather](https://t.me/BotFather)

### Установка FFmpeg

- **macOS:** `brew install ffmpeg`
- **Ubuntu/Debian:** `sudo apt install ffmpeg`
- **Windows:** скачать с [ffmpeg.org](https://ffmpeg.org/download.html) и добавить в PATH

## Установка и запуск

```bash
cd "YouTube Download"
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# В .env указать BOT_TOKEN=твой_токен
python main.py
```

## Функционал

- Кнопка **Старт** и меню (Скачать видео/аудио, Как пользоваться).
- Пользователь отправляет ссылку на видео или плейлист → бот предлагает **видео** или **только аудио**, затем **качество**.
- Отправка файла в чат (видео — документ, аудио — аудио). В подписи — ссылка на источник.
- Показ процесса: «Ищу…», «Скачиваю…», при ошибках — текст с описанием.
- Удаление служебных сообщений бота; в чате остаются только итоговые файлы/сообщения со ссылкой.

## Ограничения и возможные проблемы

- **Лимит Telegram:** бот не может отправить файл больше **50 МБ**. Для таких случаев бот пишет сообщение и оставляет ссылку на источник.
- **Плейлисты:** обрабатывается **первое видео** из плейлиста. Полная загрузка всего плейлиста возможна как доработка (по одному видео с отправкой по мере готовности).
- **Блокировки/ограничения YouTube:** используются возможности **yt-dlp** (актуальные обходы). При блокировке по региону, авторским правам или из-за изменений на сайте бот выводит понятное сообщение об ошибке.
- **YouTube недоступен в регионе (например РФ):** задай в `.env` переменную **`YTDL_PROXY`** — прокси, через который yt-dlp ходит к YouTube. Примеры: `YTDL_PROXY=http://127.0.0.1:7890` (локальный VPN/прокси) или `YTDL_PROXY=socks5://user:pass@host:1080`. Лучший вариант для 24/7 — **разместить бота на VPS за рубежом** (Европа, США): тогда к YouTube идёт доступ с сервера, и прокси не нужен.
- **Тяжёлые/длинные видео:** скачивание идёт на сервер бота; при большом размере файла пользователь получит только ссылку на источник (см. лимит 50 МБ).

## Автономная работа и развёртывание

Чтобы бот работал постоянно и можно было легко вносить изменения:

1. **VPS (рекомендуется)**  
   Арендовать сервер **вне РФ** (например, Hetzner, DigitalOcean, EU), чтобы YouTube был доступен без VPN. Либо сервер в РФ — тогда в `.env` обязательно указать **`YTDL_PROXY`** (прокси с доступом к YouTube). На сервере:
   - установить Python 3.10+, FFmpeg;
   - склонировать репозиторий, настроить `.env`, установить зависимости;
   - запускать бота через **systemd** или **screen**/tmux.

2. **Пример unit для systemd** (Linux):

   Создать файл `/etc/systemd/system/yt-dl-bot.service`:

   ```ini
   [Unit]
   Description=YouTube Download Telegram Bot
   After=network.target

   [Service]
   Type=simple
   User=ubuntu
   WorkingDirectory=/home/ubuntu/YouTube-Download
   ExecStart=/home/ubuntu/YouTube-Download/venv/bin/python main.py
   Restart=always
   RestartSec=10
   EnvironmentFile=/home/ubuntu/YouTube-Download/.env

   [Install]
   WantedBy=multi-user.target
   ```

   Затем:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable yt-dl-bot
   sudo systemctl start yt-dl-bot
   ```

3. **Обновление кода:**  
   На сервере: `git pull`, перезапуск сервиса:  
   `sudo systemctl restart yt-dl-bot`

4. **Альтернативы:**  
   Можно развернуть бота на **Railway**, **Render**, **Fly.io** (нужен Docker/образ с Python и FFmpeg) — тогда бот будет работать 24/7 без своего VPS.

## Переменные окружения (.env)

| Переменная        | Описание |
|-------------------|----------|
| `BOT_TOKEN`       | Токен бота от BotFather (обязательно) |
| `DOWNLOADS_DIR`   | Папка для временных загрузок (по умолчанию `./downloads`) |
| `MAX_FILE_SIZE`   | Макс. размер файла для отправки в чат в байтах (по умолчанию 50 МБ) |
| `YTDL_PROXY`      | Прокси для yt-dlp (если YouTube заблокирован в регионе): `http://...` или `socks5://...` |
| `UPLOAD_TIMEOUT`  | Таймаут отправки файла в Telegram в секундах (по умолчанию 300) |
